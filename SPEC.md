# 算盘桌面端规格

## 1. 当前定位

算盘是面向 macOS 的个人 A 股研究终端。当前目标是落地一个可运行的 Electron 桌面端，用于行情查看、K 线观察、自选股管理、公式编辑、条件/公式选股、研究计划和 Codex 辅助分析。

本规格描述当前可实现、可验证的产品范围。`DESIGN.md` 是视觉、术语和产品边界的来源；当两者冲突时，以 `DESIGN.md` 为准。

## 2. 产品边界

当前只包含研究和观察能力：

- 行情 / 自选 / 公式 / 选股 / 计划五个主面板。
- 全局 AI 智库侧栏和数据 / Codex 状态弹窗。
- 本地用户状态：自选股、分组、公式、选股策略、选股结果、选股历史、研究计划、AI 历史和共识。
- 真实/延迟行情优先；失败时保留本地/缓存数据。
- Codex 只能读取应用注入的数据快照和只读数据入口。

当前不包含：

- 券商账户、持仓、委托、下单、撤单或自动执行。
- 纸面账户、虚拟账户或模拟盘界面。
- Level-2 全量盘口。
- 商业行情授权分发。
- 移动端或 Web 营销页。
- React/Vite/SQLite 迁移，除非用户明确要求。

## 3. 技术栈

- 桌面壳：Electron main / preload。
- 渲染层：静态 HTML、CSS、原生 JavaScript。
- 图表：Canvas K 线和指标图。
- 主进程服务：行情刷新/导入、用户状态持久化、分析服务、Codex 数据快照。
- 本地存储：Electron `userData` 下的 JSON 文件和浏览器 localStorage 兼容层。
- 运行依赖：当前机器使用全局 Electron；新机器需要安装 Electron 或设置 `ELECTRON_BIN`。

## 4. 信息架构

底部固定五个 Tab：

1. 行情：全市场列表、搜索、排序、个股详情、K 线、指标、报价统计。
2. 自选：自选分组、批量导入、搜索、个股详情。
3. 公式：公式库、公式编辑、校验、单标的测试、批量导入。
4. 选股：内置条件、公式选股、策略保存、结果过滤、加入自选、生成研究计划。
5. 计划：研究计划列表、搜索、状态切换、打开个股、删除。

全局区域：

- 顶部栏：全 A 加载、行情刷新、行情导入、数据状态。
- 右侧 AI 智库：当前上下文、AI 状态、聊天、快速分析。
- 数据 / Codex 状态弹窗：行情、本地状态、Codex 数据入口、AI 状态。

## 5. 行情数据

行情状态必须使用统一术语：

- `真实/延迟数据`：主进程成功接入外部行情源后显示。
- `本地/缓存数据`：外部行情失败、离线或仅有本地样本时显示。

行情能力：

- 支持加载 A 股报价列表。
- 启动后优先自动加载全 A 真实/延迟报价；全 A 源限流或失败时降级刷新核心股票池真实/延迟报价，再失败才保留本地/缓存数据。
- 支持刷新当前行情和加载全 A 报价。
- 支持 JSON/CSV 行情或 K 线导入。
- 支持每只股票的日 K、分时占位绘制、MA、BOLL、MACD、KDJ、RSI；打开个股详情时按需补该股真实/延迟日 K。
- 个股资料卡只能展示当前行情快照中可证明的字段，不得伪造盘口。

## 6. 自选和本地状态

用户本地状态必须可持久化并可注入 Codex：

- 自选代码和自选分组。
- 公式库。
- 选股策略、结果和历史。
- 研究计划。
- AI 历史和 AI 共识。
- 市场排序和侧栏宽度等 UI 状态。

本地状态保存在用户本机，不展示内部文件路径给普通 UI。

## 7. 公式和选股

公式 DSL 支持白名单语法，不使用动态代码执行：

- 字段：`O H L C V AMOUNT`
- 函数：`MA EMA REF CROSS HHV LLV COUNT SUM MAX MIN ABS IF SMA STD`
- 运算：`+ - * / > >= < <= = != AND OR NOT`
- 赋值：`MA5 := MA(C,5);`
- 输出：`XG: CROSS(MA5,MA(C,20));`

选股能力：

- 内置条件：涨跌幅、成交量、均线、MACD、KDJ、新高/新低、换手率、市值、PE、PB。
- 公式选股：通过公式 DSL 计算命中结果。
- 策略保存：保存当前内置条件或公式策略。
- 结果展示：显示代码、名称、价格、涨跌幅、命中原因，并可加入自选或生成研究计划。

## 8. 研究计划

研究计划是观察和复盘材料，不是订单或外部执行指令。

计划字段：

- 标的代码、名称、市场、行业。
- 来源：个股详情、选股结果或当前个股。
- 当前报价摘要。
- 计划逻辑、观察清单、失效条件。
- 观察区间、失效线、目标观察价。
- 状态：观察中、已完成。

计划文案必须使用“研究计划”“观察复盘”“失效线”等研究语言，不使用操作化下单语言。

## 9. Codex 和 AI

AI 聊天只通过本地 Codex CLI 执行路径。Codex 不可用时，界面只提示不可用和已准备的数据范围，不生成另一套本地 AI 回复。

Codex 数据要求：

- 每次聊天都注入紧凑上下文。
- 完整数据快照写入 `codex-data/latest-context.json`。
- Codex 进程通过 `--add-dir` 获得快照目录读取权限。
- 高权限 sandbox 下可使用本机只读数据入口读取行情、自选、公式、选股结果、研究计划、AI 历史/共识和最新上下文。
- 自选股、公式库必须同时进入紧凑上下文、完整快照和只读数据范围。
- API Key 和 Base URL 仅用于 Codex 运行环境，不得写入 Codex 数据快照。

用户可见术语：

- `Codex可用`
- `Codex不可用`
- `已接入当前数据`
- `已接入应用数据`
- `真实/延迟数据`
- `本地/缓存数据`

用户界面不得展示内部地址、Header、token、内部服务名或未授权能力。

## 10. 验收标准

当前桌面端至少满足：

- 可以启动 macOS Electron App。
- 只显示五个主面板：行情 / 自选 / 公式 / 选股 / 计划。
- 顶部栏不会被 macOS 左上角窗口按钮遮挡。
- 行情列表、搜索、排序、个股详情和 K 线可用。
- 鼠标移动到图表时显示十字光标。
- 自选、公式、选股策略、结果、历史、研究计划和 AI 共识可本地持久化。
- 行情状态能区分真实/延迟数据和本地/缓存数据。
- 启动后行情列表应优先接入全 A 真实/延迟报价；外部全 A 源不可用时至少接入核心股票池真实/延迟报价。打开个股详情后应能按需补真实/延迟日 K。
- Codex 状态能说明 Codex 是否可用、API Key/Base URL 是否已配置、当前应用数据是否已接入。
- Codex 可读取自选股和公式库；这些数据可通过完整快照和只读数据范围验证。
- AI 回复不泄露内部地址、Header 或 token。
- 不出现纸面账户、虚拟账户、券商账户、下单、撤单或自动执行界面。
- 公式执行不使用 `eval`、`new Function` 或动态代码执行。

## 11. 验证命令

有意义的 UI 或运行时变更后执行：

```bash
npm run check
npm run doctor:runtime
npm run test:electron-smoke
npm run test:electron-persistence
```

窄 PATH 环境也应能找到本机 Codex 和 Electron：

```bash
PATH=/usr/bin:/bin /opt/homebrew/bin/node scripts/doctor-runtime.cjs
```
